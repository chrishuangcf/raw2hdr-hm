## üîé **SDR vs HDR: See the Difference**

When you export an HDR image at a **higher nit value** (for example 1600 or 2000 nits), the image is given **more ‚Äúheadroom‚Äù for highlights**. That means the very bright parts get extra space at the top of the HDR range so they don‚Äôt clip.

But here‚Äôs what this does to the histogram:

* The histogram shows how bright the pixels *appear* on your screen.
* If you reserve more space for highlights, the **whole curve moves left** ‚Äî meaning the image looks darker on screens that cannot reach that brightness.

**Simple idea:**
More highlight headroom ‚Üí histogram shifts left ‚Üí image looks darker.

This is normal HDR behavior.

You aren‚Äôt losing brightness ‚Äî you‚Äôre gaining more room for bright detail to exist without blowing out.

* Standard Dynamic Range (SDR) shows limited brightness on a HDR capable display.
* HDR reveals the full highlight and shadow detail your camera captured.

## **Currently this is a beta version**

* Technically, most LUTs are designed for Rec.709, which typically operates in an 8-bit SDR colorspace. To apply these LUTs correctly, the RAW image would need to be converted down into an 8-bit RGB format before the LUT can be used.

Because of this limitation, LUTs wouldn‚Äôt produce real HDR effects‚Äîthey would only serve as a creative ‚Äúfun‚Äù filter rather than an HDR-grade color tool‚Äîso this feature is not enabled at this time.
